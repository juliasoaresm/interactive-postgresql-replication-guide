Guia de Replicação Streaming PostgreSQL

1. Objetivo

Este documento fornece um guia passo a passo para configurar uma replicação streaming do PostgreSQL. O objetivo é criar um "Banco de Dados Espelho" (Servidor Réplica) que seja uma cópia em tempo real do banco de dados principal (Servidor Primário), utilizado pela plataforma ThingsBoard.

2. Requisitos Iniciais (Pré-requisitos)

Antes de iniciar a instalação, garanta que você atende aos seguintes requisitos:

2.1. Infraestrutura:

Dois Servidores: Duas máquinas (físicas ou virtuais) distintas.

Sistemas Operacionais abordados no guia: Linux (baseado em Debian/Ubuntu) ou Windows.

Acesso de Administrador: Permissão para instalar softwares e editar configurações em ambos os servidores.

2.2. Conectividade de Rede:

Os dois servidores devem estar na mesma rede e conseguir se comunicar (um consegue "pingar" o outro). A porta padrão do PostgreSQL, TCP 5432, será configurada no firewall mais adiante.

3. PASSO 1: Instalação dos Softwares

Nesta etapa, vamos instalar tudo que é necessário em ambos os servidores.

3.1. Instalação do ThingsBoard (Apenas no Servidor Primário)

Instale o ThingsBoard no servidor que você designou como Primário. Para instruções detalhadas, consulte a documentação oficial do ThingsBoard.

Instalação via Windows - https://thingsboard.io/docs/user-guide/install/windows/

Instalação via Linux - https://thingsboard.io/docs/user-guide/install/ubuntu/

(Seguindo a instalação LINUX, nela mesma já tem a instalação do postgresql e você pode pular para o item “3.3.” deste documento. Se optar pelo Windows, siga ao próximo passo.).

OBS: SEMPRE anote suas senhas e para onde elas servem.

3.2. Instalação do PostgreSQL

Instale o PostgreSQL em ambos os servidores (Primário e Réplica).

Importante: Use a mesma versão principal do PostgreSQL em ambas as máquinas.

Instruções para Windows:

Acesse o site oficial: https://www.postgresql.org/download/windows/

Baixe e execute o instalador.

Durante a instalação, ele pedirá para você definir uma senha para o superusuário postgres. CRIE UMA SENHA FORTE, ANOTE-A E GUARDE-A. VOCÊ PRECISARÁ DELA.

!!LEMBRE!! SEU POSTGRES DA MÁQUINA PRIMÁRIA TEM QUE ESTAR NA MESMA VERSÃO DO POSTGRES DA MÁQUINA DA RÉPLICA.

3.3. Instalação do DBeaver (Altamente Recomendado)

Instale o DBeaver em pelo menos uma das máquinas (recomendado: máquina primária) para facilitar a verificação dos dados.

Download: https://dbeaver.io/download/

4. PASSO 2: Coleta de Informações e Preparação

Agora que tudo está instalado, vamos coletar e definir as informações que usaremos.

Sua Ficha de Informações (Preencha agora):
    • IP do Servidor Primário - _________________________
      IP do Servidor Réplica - _________________________
      Nome para o Usuário de Replicação (Sugestão da autora: replicador) - _________________________
      Senha para o Usuário de Replicação - _________________________ (crie uma senha forte agora e anote)
      Senha do Superusuário postgres do Primário - _________________________ (a senha que você definiu no passo 3.1)

    • Como encontrar o IP de cada servidor?

    No Linux: Abra o terminal e digite "hostname -I". (Sem as aspas e o I tem que ser maiúsculo).

    No Windows: Abra o Prompt de Comando (CMD) e digite "ipconfig" (Sem as aspas e procure por "Endereço IPv4").
